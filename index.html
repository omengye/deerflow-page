Core Ecosystem Components

Connecting backend services, command-line interfaces, and mobile clients for full-scenario agent execution.

Core Backend

deerflow-api

High-performance backend service built on FastAPI and DeerFlow harness, exposing capabilities as REST and SSE API. Supports external ACP agent integration and WSL2 sandboxing for safe isolation.

  • Expose standard REST endpoints and SSE (Server-Sent Events) live streams
  • Support external ACP agents (e.g. Codex or Claude Code) via Agent Client Protocol
  • Built-in LocalWslProvider (WSL2 sandbox) for Windows compatibility and VM-level isolation
GitHub: omengye/deerflow-api
Terminal UI

deerflow-tui

A native terminal user interface client built using Go and Bubble Tea. Connected via AG-UI protocol, it delivers highly responsive real-time streaming events and keyboard-centric interaction.

  • Stream rendering of text, reasoning/thinking paths, and full tool execution lifecycles
  • Quick commands: create thread (/new), cancel run (/cancel), and exit
  • Auto-adaptive terminal width, smart wrapping, and Ctrl+C clipboard copy support
GitHub: omengye/deerflow-tui
Mobile App

deerflow-app

Native Android client built with Kotlin and Jetpack Compose. Re-implementing the state machine reducer, it offers thread history database, markdown rendering, and background SSE stream keep-alive.

  • Select local documents or capture photos directly via camera, auto-mapping to virtual workspace
  • Integrated SseForegroundService to prevent Android process limits and keep stream syncing active
  • Full Markdown rendering support including headers, lists, code blocks, and tables
GitHub: omengye/deerflow-app

Data Pipeline Architecture

The design of DeerFlow-Api architecture centers on real-time stream delivery and centralized state persistence:

Standard AG-UI Stream Delivery

TUI and Android App communicate with deerflow-api using the standard AG-UI protocol. Streaming events (text, reasoning, tool states) are pushed instantly via HTTP/SSE, guaranteeing a zero-latency interactive feel.

DeerFlow-Api Harness Orchestration

The backend encapsulates the Agent harness capability, creating a flexible execution context. Enabled with plan_mode by default, it tracks task steps and manages concurrent sub-agents efficiently.

SQLite Persistence & Thread Sharing

All session history and checkpoint states are saved in SQLite. Threads endpoints allow clients to fetch historic conversations, so you can start a task on TUI and seamlessly continue it on your phone.
